# Contributing

We welcome contributions from **all sources**, including human developers, automated systems, and AI assistants. No contribution is too small — documentation fixes, code improvements, tests, and feature ideas are all appreciated.

## How to Contribute

1. **Fork the repository** and create a feature branch.  
2. **Make focused changes** that follow the project’s style and structure.  
3. **Open a pull request** describing:
   - What you changed  
   - Why you changed it  
   - (Optional) Whether the contribution was AI‑assisted  

## Guidelines

- Keep changes clear, relevant, and easy to review.  
- Ensure code compiles and tests pass.  
- Avoid large, unsolicited refactors.  
- Maintain respectful and constructive communication.

## License

By contributing, you agree that your work will be released under the project’s existing license.
